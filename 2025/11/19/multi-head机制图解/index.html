前言

本文旨在从零开始梳理 Self-Attention 到 Multi-Head Attention 的演变过程，重点剖析各矩阵维度的扩展与变换，为 vLLM 等高性能推理框架的算子优化研究打下基础。

一、基础回顾：Scaled Dot-Product Attention (SDPA)

这是所有 Attention 机制的基石。

图中的原始输入是’Thinking Maching’，我们以 ‘Thinking’ 这个 Token 为例。在 Embedding 过程中，它被映射为一个向量。图中所示的向量维度是 $1 \times 4$，那么这里的向量长度 4 维，就代表了模型的隐藏层维度 ($D{hidden}$)。
在Multi-Head Attention拆分之前，输入维度（X）和目标维度（Q,K,V）保持一致，这样才不会丢失信息。（图绘制的时候目标维度变成了3，其实不太合理，用图看一下矩阵乘的过程就行）
输入矩阵 $X$（维度：$B \times N \times D{hidden}$）乘以权重矩阵 $W$（维度：$D{hidden} \times D{hidden}$）时，最终得到的 $Q, K, V$ 矩阵的维度仍然是：

模型在一次前向运算/后向传播里面，会同时处理 Batch Size 个输入序列。
假设 Batch Size 为 $B$，序列长度为 $N$，隐藏层维度为 $D_{hidden}$。

* 输入 $X \in \mathbb{R}^{B \times N \times D_{hidden}}$。
* 通过线性变换（权重 $W^Q, W^K, W^V \in \mathbb{R}^{D_{hidden} \times D_{hidden}}$）得到 $Q, K, V$。
  • 计算维度：
    • Score 矩阵 $S = Q K^T$ 的维度：$B \times N \times N$（$N \times N$ 即为性能瓶颈所在）。
    • 输出矩阵 $O$ 的维度：$B \times N \times D_{hidden}$。

二、Multi-Head Attention (MHA)：维度拆分与并行

Multi-Head 机制引入了 Head 数量 $H$ 和 Head 维度 $d$。

2.1 维度拆分（核心步骤）

  • 目标： 将 $Q, K, V$ 沿最后一个维度 ($D_{hidden}$) 拆分成 $H$ 个 Head。
  • 维度变化图解： 详细展示从三维到四维的变换。
    • 其中，$d = D_{hidden} / H$。

2.2 并行计算与结果融合

  • 并行计算： 在 $H$ 个 Head 维度上独立、并行地执行 SDPA 运算。
    • $H$ 个独立的 Attention Blocks 同时运行。
  • 结果融合： 将 $H$ 个 Head 的输出结果 $O_1, \dots, O_H$ 重新拼接起来（Concatenate），然后通过最终的线性投影层。
    • 维度变化： $[B, H, N, d] \xrightarrow{\text{Concatenate}} [B, N, D{hidden}] \xrightarrow{\text{Projection}} [B, N, D{hidden}]$。

三、性能优化：Q/K/V Head 数量的差异化处理

针对推理场景（vLLM），MHA 的优化变体是研究重点。

3.1 GQA/MQA 的引入背景

  • 问题： MHA 中 $K, V$ 的 Head 数量与 $Q$ 相同，导致 KV Cache 占据大量内存。
  • 解决思路： 减少 $K, V$ 的 Head 数量，让多个 $Q$ Head 共享同一组 $K, V$ Head。

3.2 三大 Head 数量关系的对比（图示）

策略 Q Head 数 ($H$) K/V Head 数 ($H_{kv}$) 关系 ($H:H_{kv}$) 主要优势
MHA $H$ $H$ $1:1$ 丰富的特征多样性
GQA $H$ $1 < H_{kv} < H$ $G:1$ (分组共享) 平衡性能与内存节省
MQA $H$ $1$ $H:1$ (完全共享) 最大限度节省 KV Cache 内存

四、总结与 vLLM 算子优化关联

  • 核心要点： Multi-Head 机制通过维度拆分实现了并行化，而 GQA/MQA 通过维度不对称实现了内存优化。
  • 工程挑战： 在 vLLM/FlashAttention 中，算子必须能够高效、灵活地处理 不对称维度（GQA/MQA）下的 维度拆分、重组与并行。这是实现高性能 Prefill 和 Decode 阶段的关键。
